Workforce Analytics · Attrition & Retention

Operational efficiency deep dive by Angelika Grzybowska – understanding why employees leave and how to improve retention

As an HR data analyst, I built this project to uncover drivers of attrition. Using employee records, termination data, and department hierarchies, I analysed turnover trends, tenure, and manager impact. All data synthetically generated but mirrors real‑world patterns.

Key analyses & SQL

Monthly attrition rate

Measure turnover over time (resignations).

View SQL 
WITH months AS (
    SELECT DISTINCT strftime('%Y-%m', hire_date) AS month FROM employees
    UNION
    SELECT DISTINCT strftime('%Y-%m', termination_date) FROM terminations
),
headcount AS (
    SELECT strftime('%Y-%m', hire_date) AS month,
           COUNT(*) AS cnt
    FROM employees
    WHERE hire_date <= date('now')
    GROUP BY strftime('%Y-%m', hire_date)
),
attritions AS (
    SELECT strftime('%Y-%m', termination_date) AS month,
           COUNT(*) AS cnt
    FROM terminations
    WHERE termination_date IS NOT NULL
    GROUP BY strftime('%Y-%m', termination_date)
)
SELECT m.month,
       ROUND(COALESCE(a.cnt, 0) * 100.0 / NULLIF(COALESCE(h.cnt, 1), 0), 1) AS attrition_rate
FROM months m
LEFT JOIN headcount h ON m.month = h.month
LEFT JOIN attritions a ON m.month = a.month
ORDER BY m.month;
Insight: Attrition spikes in Q1 2024 (9.2%) – possibly linked to annual bonus cycle.

Attrition by department

Which departments lose the most employees?

View SQL 
SELECT d.name AS department,
       COUNT(e.id) AS total_employees,
       COUNT(t.employee_id) AS terminated,
       ROUND(COUNT(t.employee_id) * 100.0 / COUNT(e.id), 1) AS attrition_pct
FROM departments d
LEFT JOIN employees e ON d.id = e.department_id
LEFT JOIN terminations t ON e.id = t.employee_id
GROUP BY d.id
ORDER BY attrition_pct DESC;

Tenure at termination

How long before leaving?

SQL 
SELECT
    CASE
        WHEN (JULIANDAY(t.termination_date) - JULIANDAY(e.hire_date)) / 365 < 0.5 THEN '<6 months'
        WHEN (JULIANDAY(t.termination_date) - JULIANDAY(e.hire_date)) / 365 < 1   THEN '6–12 months'
        WHEN (JULIANDAY(t.termination_date) - JULIANDAY(e.hire_date)) / 365 < 2   THEN '1–2 years'
        WHEN (JULIANDAY(t.termination_date) - JULIANDAY(e.hire_date)) / 365 < 3   THEN '2–3 years'
        ELSE '3+ years'
    END AS tenure_bucket,
    ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM terminations), 1) AS pct_leavers
FROM employees e
JOIN terminations t ON e.id = t.employee_id
GROUP BY tenure_bucket
ORDER BY MIN(JULIANDAY(t.termination_date) - JULIANDAY(e.hire_date));

Hiring cohort survival

Retention by hire quarter.

SQL 
SELECT
    strftime('%Y-Q', e.hire_date) || strftime('%q', e.hire_date) AS cohort,
    ROUND(COUNT(CASE WHEN t.employee_id IS NULL THEN 1 END) * 100.0 / COUNT(*), 1) AS retention_rate_1y
FROM employees e
LEFT JOIN terminations t ON e.id = t.employee_id
    AND JULIANDAY(t.termination_date) - JULIANDAY(e.hire_date) <= 365
WHERE strftime('%Y', e.hire_date) IN ('2023', '2024')
GROUP BY cohort
ORDER BY cohort;

Attrition by manager

Managers with high turnover.

SQL 
SELECT
    m.name AS manager,
    COUNT(e.id) AS team_size,
    COUNT(t.employee_id) AS leavers,
    ROUND(COUNT(t.employee_id) * 100.0 / COUNT(e.id), 1) AS attrition_pct
FROM employees m
JOIN employees e ON m.id = e.manager_id
LEFT JOIN terminations t ON e.id = t.employee_id
GROUP BY m.id
HAVING COUNT(e.id) >= 5
ORDER BY attrition_pct DESC
LIMIT 5;

Salary impact

Do lower salaries drive attrition?

SQL 
SELECT
    CASE
        WHEN e.salary < 50000 THEN '<50k'
        WHEN e.salary < 75000 THEN '50-75k'
        WHEN e.salary < 100000 THEN '75-100k'
        ELSE '>100k'
    END AS salary_band,
    COUNT(*) AS total_employees,
    COUNT(t.employee_id) AS leavers,
    ROUND(COUNT(t.employee_id) * 100.0 / COUNT(*), 1) AS attrition_pct
FROM employees e
LEFT JOIN terminations t ON e.id = t.employee_id
GROUP BY salary_band
ORDER BY MIN(e.salary);
Under $50k: 28% attrition – review junior comp.
